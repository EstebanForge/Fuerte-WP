== Description ==

🛡️ **Professional WordPress Security & Maintenance That Prevents Problems Before They Happen**

Every day, WordPress sites are compromised through supply chain attacks - when even trustworthy developers have their accounts hacked and malicious code pushed as "updates." Fuerte-WP protects you when developers can't protect their own update systems.

**🚨 CRITICAL SUPPLY CHAIN PROTECTION**

Supply chain attacks occur when attacker's compromise developer accounts and push malicious updates that thousands of sites auto-install within hours. Fuerte-WP's **Blocked Updates** feature lets you freeze plugins at safe versions instantly when you learn of attacks, protecting your site while others are compromised.

*The developer is a victim too. Their credentials were stolen, systems breached. Fuerte-WP gives you control when even the best developers can't protect their own update systems.*

**⚡ SMART AUTO-UPDATE MANAGEMENT**

* **Scheduled Updates**: Choose update frequency (6, 12, 24, or 48 hours)
* **Selective Control**: Enable/disable auto-updates for core, plugins, themes, translations separately
* **Deferred Updates**: Exclude specific plugins from auto-updates while keeping manual control
* **Blocked Updates**: Completely freeze compromised plugins during supply chain attacks
* **Zero Performance Impact**: Background processing doesn't slow down your site

== Installation ==

1. Install Fuerte-WP from the WordPress plugin directory
2. Activate the plugin
3. Visit Settings > Fuerte-WP to configure settings
4. Add your email as a super user (you'll bypass all restrictions)
5. Set up your auto-update preferences
6. You're protected!

== Frequently Asked Questions ==

= How does supply chain attack protection work? =

When you learn that a plugin developer's account has been compromised and malicious updates are being distributed, you can immediately block that plugin in Fuerte-WP. This freezes the plugin at its last safe version and prevents your site from auto-installing malicious code, even while thousands of other sites are being compromised.

= What's the difference between Deferred and Blocked updates? =

**Deferred Updates**: The plugin won't auto-update, but you can still manually update it when you're ready. Good for testing updates before applying them.

**Blocked Updates**: The plugin cannot update at all - neither automatically nor manually. Essential during supply chain attacks when any update could contain malicious code.

= Can other administrators disable Fuerte-WP? =

No! Only super users can modify Fuerte-WP settings or disable the plugin. Other administrators are restricted from making changes that could compromise your site's security.

= Will this slow down my website? =

Absolutely not. All maintenance tasks run in the background and don't affect page load times. Fuerte-WP is optimized for performance.

= Does this work with WordPress multisite? =

Yes! Fuerte-WP is fully compatible with WordPress multisite installations and can be network-activated for centralized management.
